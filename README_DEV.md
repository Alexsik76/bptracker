# Локальне середовище розробки (Development Environment)

Цей документ описує процес запуску та налаштування локального середовища розробки для проекту BP Tracker.

---

## 1. Запуск бази даних та бекенду (Docker)

Бекенд разом із базою даних PostgreSQL запускаються у контейнерах Docker за допомогою спеціальної конфігурації для розробки.

### Крок 1: Перейдіть у директорію бекенду
```bash
cd bptracker-backend
```

### Крок 2: Запустіть контейнери
Виконайте команду для збірки та запуску в режимі розробки:
```bash
docker compose -f docker-compose.dev.yml up --build -d
```
Це запустить:
- Базу даних PostgreSQL на порту `5436`.
- Бекенд API на порту `5000` (`http://localhost:5000`).

На старті бекенд автоматично:
- Застосує всі міграції бази даних.
- Запустить сідер даних, який створить дев-користувача `dev@local` (ID: `e6915d20-30fd-4154-b43a-85c04dbac190`), активну схему лікування, шаблон нагадувань та кілька вимірювань тиску.

---

## 2. Запуск фронтенду (Node.js локально)

Фронтенд запускається локально за допомогою Vite.

### Крок 1: Перейдіть у директорію фронтенду
```bash
cd ../bptracker-frontend
```

### Крок 2: Встановіть залежності (якщо запускаєте вперше)
```bash
npm install
```

### Крок 3: Запустіть сервер розробки
```bash
npm run dev
```
Фронтенд буде запущено (зазвичай на `http://localhost:5173`). Усі запити до `/api` будуть автоматично проксуватися на бекенд (`http://localhost:5000`).

---

## 3. Авторизація у режимі розробки (Dev Auth Bypass)

Щоб увійти під тестовим користувачем `dev@local` з реальними даними із бази даних:

1. Відкрийте ваш додаток у браузері (наприклад, `http://localhost:5173`).
2. Відкрийте Інструменти розробника (F12) -> Консоль (Console).
3. Виконайте наступний JS-код для швидкої авторизації:
   ```javascript
   fetch('/api/v1/auth/dev-login', { method: 'POST' })
     .then(res => res.json())
     .then(data => {
       console.log('Успішний вхід:', data);
       location.reload(); // Перезавантажити сторінку для оновлення стану
     })
     .catch(err => console.error('Помилка авторизації:', err));
   ```
4. Після перезавантаження сторінки ви будете авторизовані як `dev@local`, а інтерфейс відобразить усі попередньо згенеровані вимірювання та активну схему лікування.

---

## 4. Корисні команди для розробки

### Перегляд логів бекенду
```bash
cd bptracker-backend
docker compose -f docker-compose.dev.yml logs -f api
```

### Перезапуск API контейнера (наприклад, після локальних змін у коді для перезбірки)
```bash
cd bptracker-backend
docker compose -f docker-compose.dev.yml restart api
```

### Повна зупинка середовища розробки
```bash
cd bptracker-backend
docker compose -f docker-compose.dev.yml down
```
